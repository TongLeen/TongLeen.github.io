Part-2 函数计算的硬件实现
===

这一部分的代码位于 [GitHub](https://github.com/TongLeen/SoC-Course-Design/tree/main/Part2)


## 设计需求

本次实验要求分为两部分：
1. 设计一个复杂函数的计算单元，要求可以计算正弦函数、余弦函数、平方根和对数函数；
2. 设计UART通信协议与上位机，使得上位机可以通过UART发送数据，实验FPGA计算并通过UART返回结果。

## 设计方案

### 计算单元的设计

为了计算的通用性，设计的输入和输出均为 IEEE754 定义的标准 32bit 浮点数，适应 CPU 常用的 float32 数据类型。

复杂函数的计算将通过查表实现。包括一个正弦表、一个平方根表和一个指数表。

每个表存储 4096 个数据(12bit 地址)，表示输入值的范围为 $0 \sim \pi/2$ ，则每两个数据点之间的步长为
$$\frac{\pi}{2}/4096 \approx 3.8 \times 10^{-4}$$
小于1‰。每个数据点 18bit ，数据格式为自行设计的压缩浮点表示。

#### 查找表的设计

查找表的设计是考虑到 FPGA 中的 9k BRAM 的参数。

每个 9k BRAM 的位宽是 18bit，可存储 512 个 18bit 数据。将数据位宽设计为 18bit 可以减小 BRAM 存储空间的浪费。
数据点数为 4096 是 512 的整数倍，可以更好地利用 BRAM 资源。

每个表将会占用 8 个 9k BRAM，共使用 24 个 BRAM，FPGA 一共有 64 个 9k BRAM，FPGA 中的 BRAM 资源依旧充足。

##### 表地址的映射

考虑表的大小与函数的特性，表的地址映射为$0 \sim \pi/2$，是为了适配三角函数。有以下几点考虑：
1. 正弦和余弦函数具有对称性，单个象限内的数据即可实现对完整周期数据的重建，因此只需要保存单个象限的数据即可；
2. 正弦和余弦函数具有周期性，如果让单个周期内对于的地址范围是 2 的整数倍，即可简化去重复周期的操作，可以依赖二进制码的周期性，直接截断高位；
3. 为了适配三角函数的设计，简化地址译码结构，将平方根和指数函数表的范围同样设定为$0 \sim \pi/2$，格式统一。

每个表的数据点数量已知同样是为了简化译码结构，选取4096个点是有以下考虑：
1. 平方根函数在0附近导数无穷大，这个位置误差较大无法避免；
2. 正弦和余弦导数的最大值是1，即输出的误差最大就是输入误差，为了尽量获得较高精度，选取4096个点，这样在输出误差最大的情况下也只有$3.8 \times 10^{-4}$；
3. 指数函数在整个范围内最大值为$\exp(\pi/2) \approx 4.81$，这使得最大输出的误差为$1.8 \times 10^{-3}$，略微超过1‰，勉强可接受。

##### 表中的数据格式

查找表中数据的格式，其位长为 18bit，可以直接映射为 32bit 的浮点数，设计过程如下：
1. 表数据不保存为 32bit 浮点是因为值的范围较小，32bit 存在大量冗余，数据位长为 18bit 是为了适应 BRAM 的位宽；
2. 在$0 \sim \pi/2$范围内，正弦表的值为$0 \sim 1$，平方根表的值为$0 \sim 1.2533$，指数表的值为$1 \sim 4.81$，不同函数的输出范围相差较大，因此在设计数据时归一化，存储平方根值的$1/2$，指数值的$1/8$，不过不用担心，放缩不会影响存储数据的精度；
3. 由于平方根表在 0 附近的值误差较大，仅考虑正弦表在 0 附近的精度，0 附近正弦表相邻两点的距离为$3.8 \times 10^{-4}$，这是表中存储的非 0 最小值；
4. 根据 IEEE754 的定义，`1` 的浮点表示的指数位为`0b0111_1111`，$3.8 \times 10^{-4}$的浮点表示的指数位为`0b0111_0011`，那么可以忽略前 4bit，仅保留后 4bit 作为存储数据的浮点位。
5. 由于存储的值都是正数，符号位也可以不保存；
6. 剩余 14bit 用来存储浮点数的尾数的高 14bit，虽然只存储 14bit，实际有效数字是 15bit，表示的数据误差仅为$3.05 \times 10^{-5}$，比我们表示的最小的非零数据小 1 个数量级，可以接受。

#### 译码单元设计

译码部分就是要将输入的浮点数转为查找表的地址信号，数学上的过程就是将输入的浮点数乘以转换比例，在对结果取整即可。

在硬件电路设计中，浮点数乘法和浮点数转整数不能直接进行，这一步必须手动处理。处理过程如下：
1. 提取输入浮点数的符号位，指数和尾数部分；
2. 将尾数 MSB 拼接`0b1`，形成完整的底数；
3. 将得到的底数乘以一个固定的整数，这个整数由转换比例计算出来；
4. 将相乘得到的结果进行移位，根据原先的浮点数指数和转换比例；
5. 将移位后的结果的高位截断；
6. 根据符号位，将截断后的结果变为补码的负数（或保持原样）；
7. 输出最终结果。

经过这样的操作，原先的浮点数就被周期性映射到$-\pi \sim \pi$区间内，这里实际输出的数据为 14bit。这 14bit 对于正弦和余弦函数是全部有效的，而对于平方根和指数，分别取其低 12bit 和低 13bit。

数学上乘以转换比例被映射为了上面的(3)(4)两步，原理如下：
1. 乘以一个小数$n$，近似等于乘以$n \times 2^k$取整后右移$k$位，$k$越大越精确；
2. 取一个合理的$k$，计算$N = n \times 2^k$
3. 将原数乘以$N$，得到的结果为$R$，再右移$k$位得到的数据$R >> k$即为最终结果。

而我们的输入是一个浮点，因此在移位时，要加上浮点数指数位的修正。

在本方案中，转换比例为$1/\frac{\pi}{2} \times 4096 \approx 2608$，这个数有效位数是12bit，精度足够，选择$k=0$即可。


#### 地址预处理

由浮点译码来的地址是 14bit 的，因此在实际进入查找表之前需要对地址进行转换。

对于指数函数和平方根函数，可以直接截断低 12bit 直接送入表；而对于三角函数，需要根据高 2bit 地址对后 12bit 地址进行处理。

##### 三角函数的特殊处理

对于三角函数，其前高 2bit 实际表示的就是象限。而对于 2、4 象限，函数值与 1、3 象限的变化趋势刚好相反，因此当第 12bit 为`1`，即位于第 2、4 象限时，要对地址进行镜像，直接对输入的地址按位取反即可。

这里对于余弦函数的处理很巧妙，因为在数学上余弦函数可以由正弦函数向左平移一个$\pi/2$得到，因此直接在表示象限的高 2bit 加上 1 即为当前正弦表输出余弦值的设置。


#### 数据后处理

对于查表得到的数据，其实是无法直接使用的，必须先组装成 32bit 的浮点数，外加一点处理才能使用。组装这一步就按照上述过程的逆过程即可。唯一注意的是，当 18bit 中的指数部分为 0 时，表示这个数据本身为 0，此时扩展后的浮点指数也是全 0。

##### 平方根与指数的最终值

对于平方根函数，需要对输出值的指数段加一，因为在设计表时，将存储的数据值进行了除以 2 的放缩；同样的处理应用于指数函数，将指数值加三。对于三角函数这一步不需要。从这里可以看见，原数据的放缩没有影响最终有效位，不会影响到数据的精度。在这一步，平方根函数值可以输出了。

##### 三角函数的处理

三角函数需要根据输出地址的第13bit决定输出的浮点数的符号位，将浮点数最高位替换为输入地址的 13bit 即可，三角函数也处理完成，可以输出了。

##### 指数函数的处理

最后对指数函数进行处理。在表中存储的指数函数，其输入范围是$0 \sim \pi/2$，这里通过处理将其值映射到$-\pi/2 \sim 0$上。

这里的原理是这样的：对于指数函数，$\exp(x+a) = \exp(x) \times \exp(a)$，当我们得到了$0 \sim pi/2$范围内的指数函数的值，我们可以通过平移并乘以一个系数得到$-pi/2 \sim 0$范围内的函数值。

由于二进制码的循环的特性，直接取 12bit 低位即为平移后的地址。第 12bit 如果为 1，则说明该输入位于 y 轴左侧，此时需要乘以一个系数。在本设计中，两段区间相差了$\pi/2$的距离，因此负半轴的值应该被乘以$1/\exp(\pi/2) \approx 0.20788$。

现在这个问题变成了一个 32bit 浮点数乘以一个小数。可以将这个小数也变为 32bit 浮点数的格式，然后按照标准方法相乘得结果，这里不再赘述。得到的结果即为指数函数的输出。



### 上位机与UART通信数据帧设计

考虑最简单：计算单元是无状态的，FPGA 也是无状态的。每次上位机请求计算时，要同时发送要计算的函数。

这通常不是最好的做法，因为大量数据批量做一种运算时，效率会下降 25%，不过为了设计简单，每次计算时给出要计算的函数。

设计单次输入一个 32bit 浮点数，输出一个 32bit 浮点数，因此输入 5 字节，输出 4 字节，不考虑传输出错。

输入的第一个 byte 为计算的函数，`0` 为平方根，`1` 为指数，`2` 为正弦，`3` 为余弦；其余 4 字节为大端序的 32bit 浮点的二进制表示。

返回值也是 4 字节大端序的 32bit 浮点的二进制表示。

上位机依旧使用 Python + `pyerial`实现，在上位机程序中保存使用函数的状态，每次发送一个数据。


### UART传输数据帧的硬件设计

这里设计了一个串转并的模块，接受固定字节的数据，当接受数量达到特定值时，将数据并行给出。

当接收到 5 byte 的数据后，将 5 个字节并行输出，送给函数计算电路计算。

当计算完成后，计算电路将数据给到一个并转串的模块，模块将得到的数据按字节输出给 UART 发送模块。


## 设计实现

以下给出各个模块具体的接口设计和内部实现思路，以及一些设计上的细节见完整代码。

### 运算单元

运算单元使用的端口如下

| 引脚        | 方向 | 功能                     |
| ----------- | ---- | ------------------------ |
| clk         | in   | 50MHz时钟信号            |
| rst         | in   | 同步复位信号，高电平复位 |
| wdata[31:0] | in   | 数据输入                 |
| wfunc[1:0]  | in   | 函数输入                 |
| wvalid      | in   | 数据输入有效             |
| wready      | out  | 数据输入就绪             |
| rdata[31:0] | out  | 数据输出                 |
| rvalid      | out  | 数据输出有效             |
| rready      | in   | 数据接收端就绪           |


模块内部使用 2 级流水线。第一级寄存器位于浮点数转地址的输出，此时寄存器存储的是查找表的地址；第二级寄存器为查找表的输入寄存器，其输出直接到模块输出。

其内部由 4 个模块组成，首先使用浮点数转定点数模块将输入的浮点数转为 14bit 地址，分发给正弦/余弦模块、平方根模块和指数模块，然后将各个模块输出给到数据选择器，数据选择器将根据`wfunc`信号选出对应函数的结果。

各个子模块的工作方式如前文所述。

这两个模块内部的关键是一个移位寄存器。

对于串转并模块，实际上是由一端输入，同时输出的移位寄存器；并转串模块是带置位功能的移位寄存器。

每个模块内部设计一个计数器，用来管理输出逻辑，当计数达到设定值时，进行输出/停止输出。
