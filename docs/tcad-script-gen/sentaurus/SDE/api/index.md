# API 总览


本库的 API 围绕 **命令收集器（Command Collector）** 模式设计。

核心思想是：通过 `useSde<M, D>()` 工厂函数创建一个上下文对象，该对象内部维护一个 `string[]` 类型的命令列表。所有暴露的 API 方法均向此列表追加 SDE 命令字符串，直到用户调用 `save(filename)` 将全部命令一次性写入脚本文件。

API 被划分为 **四个领域模块**（Draw、Contact、Dop、Mesh），每个模块对应 SDE 工作流中的一个环节：

- **Draw** — 器件几何结构的创建与组合（矩形、多边形、布尔合并、圆角处理、坐标系设置）
- **Contact** — 电极/接触的定义与绑定
- **Dop** — 掺杂分布的定义（常量掺杂、高斯掺杂），支持按位置窗口、材料、区域三种作用范围
- **Mesh** — 网格细化控制（精细化区域定义、界面偏移加密、细化函数绑定）

这四个模块的方法在底层委托给 `core/` 层的纯函数（如 `create_rectangle`、`define_constant_profile` 等），由它们完成 SDE 命令字符串的格式化拼接。这种分层设计使得上层 API 专注于类型安全与易用性，底层函数保持纯粹的可测试性。

设计上还利用了 TypeScript 的 **泛型** 与 **可辨识联合（Discriminated Union）** 来提供编译期约束：`useSde<M, D>` 的两个类型参数 `M`（材料名）和 `D`（掺杂剂名）会贯穿所有子模块，确保用户只能传入预先声明的合法值；而 `kind` 字段作为可辨识标签，使 TypeScript 能根据其值自动收窄参数类型，在编码时提供精准的智能提示与必填字段校验。

## 使用方法

通过对象解构从 `useSde<M, D>()` 中提取各模块和 `save` 方法：

```typescript
const { draw, contact, dop, mesh, save, run, runAndExit } = useSde<M, D>();
```

- **`draw`** — 绘制几何区域（矩形、多边形、布尔合并、设置坐标模式等）
- **`contact`** — 定义电极/接触
- **`dop`** — 设置掺杂（常量分布或高斯分布）
- **`mesh`** — 控制网格细化策略（精细化区域、界面偏移加密、构建网格）
- **`save`** — 将所有积累的命令写入脚本文件
- **`run`/`runAndExit`** — 使用 SDE 执行所有积累的命令（并退出）

工作流程为：
1. 调用 `useSde<M, D>()` 创建上下文
2. 以解构出的模块描述器件结构
3. 所有命令积累完毕后调用 `save("filename")` 输出脚本
4. 使用 `run()` 调用 SDE 执行所有积累的命令；`runAndExit()` 是其运行完自动退出的快捷版本
