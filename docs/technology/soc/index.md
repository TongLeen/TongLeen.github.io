片上系统 SoC 的软硬件设计
===

> 这是一门来自**电子科技大学**的选修课


## 片上系统 SoC 是什么？

一个可用的计算机系统通常要包含 5 个抽象的设备：\
运算器、控制器、存储器、输入设备、输出设备

对于常规的计算机而言，维持其运行的物理结构至少要包括 3 个：
1. 处理器：用于执行运算任务、控制整个系统；
2. 存储器：用于存储运算过程中的；
3. 输入输出设备：与外界环境交互；

上面这三部分常规情况下都是单独的器件，因此一般的计算机通常需要多个电子器件组合而成，至少上面这三个是分立的。

当把一个计算机系统实现在一颗芯片里，一颗芯片就完成了整个计算机系统的功能，我们把这种实现方式称为**片上系统（System on Chip，SoC）**。

> 这里的 *一颗芯片* 指的是同一个 DIE 上，可以说 ASIC ，也可以是 FPGA \
> 但是并不是看起来在一颗芯片里就是 SoC ，也可以是 [SiP](https://www.baidu.com/s?wd=SiP%20系统级封装)

我们肯定不会去造ASIC，因此使用FPGA作为实验平台。

## 要实现什么内容？

作为一个位于**高等教育授课末期**的实验性课程，要求肯定不会太低。

这门课总体上要求在 FPGA 上实现一个可以运行的 SoC ，其中
1. 使用 Arm Cortex-M0 作为 SoC 的处理器，这个由课程提供；
2. 使用 AHB 总线搭建系统，自行设计存储单元和输入输出设备；
3. 为处理器编写程序，使其可以计算保留三角函数、指数函数在内的数学函数；
4. 使用硬件电路实现上述的数学函数，将其集成到系统当中；
5. 对比硬件单元与软件计算的计算速度；


整个课程分为 4 个步骤（4次课）：
1. 熟悉开发环境和平台，搭建 Verilog 和 FPGA 环境，写出串口通信模块；
2. 使用数字逻辑电路实现数学函数的计算；
3. 搭建 SoC ，搭建 Keil 开发环境，调试处理器的 IP 核，软件实现函数计算；
4. 将硬件实现的函数计算模块集成到 SoC 中，对比软硬件的计算速度；

## 实现上述任务有什么要求？

上面的内容涵盖了电子信息中关于软硬件开发的大部分内容：
| 层级     | 知识                  | 应用点                                                                                                       |
| -------- | --------------------- | ------------------------------------------------------------------------------------------------------------ |
| 器件     | 微电子器件            | 了解 CMOS 开关的特性，理解背后的时延、功耗来源                                                               |
| 硬件     | 数字逻辑与数字电路    | 数字逻辑是整个数字系统的基础，理解如何用 CMOS 开关来表示信息和状态                                           |
| 硬件     | 集成电路原理          | 理解 CMOS 开关的非理想因素在实际的集成电路中的影响，以及如何消除                                             |
| 硬件     | FPGA 的实现原理       | 了解 FPGA 内部的结构，以便于更高效地利用 FPGA 内的资源，同时清楚其构成电路的非理想特性                       |
| 硬件     | EDA 技术              | 理解如何使用硬件描述语言 HDL 来设计数字电路，其背后的原理是什么                                              |
| 硬件     | Verilog/SystemVerilog | 学习一门具体的硬件描述语言用于描述和设计数字电路                                                             |
| 硬件     | 常见通信协议          | 了解嵌入式系统中常见的通信协议，比如 UART、I2C、SPI等，用于实现系统与外部通信                                |
| 数字系统 | 微机原理与接口        | 理解一个完整的数字系统中不同功能的组件如何相互配合，实现计算机系统的功能                                     |
| 数字系统 | 计算机组成原理        | 了解流水线、不同的执行阶段以及执行指令在硬件上的表现（与计算机专业不同，我们更倾向于理解程序在硬件上的表现） |
| 数字系统 | AMBA 总线             | Arm 推动的总线架构，本次课程使用的 AHB 即为 AMBA 的一部分                                                    |
| 软件     | Arm Cortex-M0         | 本次实验实验的处理器核，需要了解其特性：复位、栈、内存布局、中断等                                           |
| 软件     | 汇编                  | 虽然大部分工作可以交给 C 完成，但是系统最初的初始化工作需要汇编                                              |
| 软件     | C 语言                | 使用 C 来完成处理器程序的编写                                                                                |
| 软件     | 编译原理              | 虽然我们不需要写编译器，但是编译原理可以大致知道一段 C 程序编译后大概对应什么样的汇编                        |
| 软件     | 二进制文件结构        | 对于编译出的结果，我们要知道不同段对应的是什么，需要将什么数据加载到内存的何处                               |
| 软件     | 一门高级语言          | 编写脚本、上位机等，处理一些杂活和文件处理                                                                   |
